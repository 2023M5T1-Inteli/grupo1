ok(): para retornar uma resposta HTTP de sucesso (código 200) com um corpo opcional.

created(): para retornar uma resposta HTTP de sucesso (código 201) com um cabeçalho Location que indica a URI do novo recurso criado e um corpo opcional.

noContent(): para retornar uma resposta HTTP de sucesso sem conteúdo (código 204).

badRequest(): para retornar uma resposta HTTP com um erro de cliente (código 400) e uma mensagem de erro opcional no corpo.

notFound(): para retornar uma resposta HTTP indicando que o recurso não foi encontrado (código 404) e uma mensagem de erro opcional no corpo.

unprocessableEntity(): para retornar uma resposta HTTP com uma mensagem de erro de validação de entrada (código 422) e uma mensagem de erro opcional no corpo.
